# Деплой на GitHub Pages

**Тема:** Оценка стоимости облачного хостинга для веб-проекта  
**Студент:** Пронин Вячеслав Константинович, ИУ5Ц-73Б  
**Ветка:** `gh-pages`

## Назначение
Ветка для автоматического деплоя фронтенд-приложения на GitHub Pages.

## Структура
- Содержит только собранные статические файлы (HTML, CSS, JS)
- Автоматически обновляется через GitHub Actions
- Используется для демонстрации работы приложения

## Технологии
- GitHub Pages
- GitHub Actions
- Vite (сборка)
- PWA (Progressive Web App)

## Как работает
1. При пуше в ветки `main`, `SPA`, `Redux` запускается GitHub Actions workflow
2. Собирается production-версия приложения
3. Статические файлы копируются в ветку `gh-pages`
4. Приложение становится доступным по адресу:  
   `https://vyacheslav-programmer.github.io/Web-project_cloud_cost_RIP_frontend_2025/`

## Особенности
- Поддержка PWA (можно установить на телефон как приложение)
- Адаптивный дизайн для мобильных устройств
- Подключение к бэкенду через прокси
- Fallback на мок-данные при недоступности бэкенда
