# Лабораторная работа 6: Redux и адаптивность

**Тема:** Оценка стоимости облачного хостинга для веб-проекта  
**Студент:** Пронин Вячеслав Константинович, ИУ5Ц-73Б  
**Ветка:** `Redux`

#### Цель работы
Внедрение адаптивности и развертывание приложения системы оценки стоимости облачного хостинга.

#### Задание
Внедрить менеджер состояний для хранения значений фильтров, добавление адаптивности и PWA, создание Tauri и развертывание в `Pages`.

Добавление менеджера состояний `Redux Toolkit` для хранения фильтра тарифов. Рекомендуется использовать расширение для браузера [redux-devtools](https://chromewebstore.google.com/detail/redux-devtools/lmhkpmbekcpmknklioeibfkpmmfibljd). Необходимо развернуть фронтенд на `GitHub Pages` и добавить возможность работы в режиме `PWA`. Добавить адаптивность для трех страниц приложения.

Создание простого нативного приложения на `Tauri` для интерфейса гостя (без авторизации и редактирования), состоящий из 3 страниц с фильтрацией и картинками. Подключить приложение к разработанному API через IP адрес в локальной сети (не `localhost`).

#### Порядок показа
1. Открыть `GitHub Pages` с mock на телефоне, сохранить PWA.
2. Открыть PWA, применить фильтрацию тарифов, перейти на главную и вернуться, чтобы показать старое значение фильтра.
3. Перейти в адаптивный режим браузера, поменять ширину.
4. Объяснить в коде настройки для размера карточек, количества колонок и тд - показать конкретные значения размера и количества колонок, карточек.
5. Открыть на компьютере `GitHub Pages`, продемонстрировать работу с бэкендом.
6. Показать работу с бэкендом собранного приложения Tauri, не dev.
7. На компьютере показать подключение к бэкенду по IP локальной сети (не `localhost`), сравнить IP сервера из консоли и в коде приложения.
8. Через `wireshark` или `tcpdump` найти порт вашего `Tauri` приложения.
9. Отредактировать тарифы в БД и продемонстрировать изменение в Tauri.
10. Показать `HTTPS` в локально развернутом фронтенде.

#### Контрольные вопросы
- Flux
- Схема redux (store, reducer, dispatch, action)
- Виды нативных приложений (PWA, Tauri и тд)
- Pages

#### Deployment диаграмма
Все узлы и компоненты системы: фронтенда, web-сервера со статикой, веб-сервиса, базы данных и других хранилищ и тд. Узлы соединить протоколами, компоненты фронтенда и бэкенда поместить в узлах, указать API между ними.
